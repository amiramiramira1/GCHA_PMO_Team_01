## Section 2: Lifecycle Choice by Site

| Component | Site | Methodology | Justification |
|-----------|------|-------------|---------------|
| **Security & Database** | Cairo | V-Model | Data sensitive, GDPR compliance, stable requirements |
| **AI Analytics Engine** | Berlin | Spiral Model | High technical risk, experimental AI, iterative approach needed |
| **Mobile Nurse Interface** | Bangalore | Agile/Scrum | User feedback important, evolving requirements, 2-week sprints |
| **PMO Coordination** | London | Hybrid Management | Oversees all sites, ensures integration |

**Rationale:**  
No single methodology fits all. Hybrid allows Cairo to be secure and compliant, Berlin to innovate safely, and Bangalore to quickly adapt to user feedback.

## Section 3: Buy vs. Build Decision

### Decision: BUILD

**Reasons:**
1. **Competitive Advantage:** AI engine is core strategic differentiator.  
2. **Customization:** Tailored to GlobalCare’s patient data and workflows.  
3. **Ownership:** Building in-house keeps IP and allows patents.  
4. **Regulatory Control:** Full control over GDPR/FDA compliance.  
5. **Long-Term Costs:** Avoids perpetual licensing fees and vendor lock-in.

### Components We Will Buy (COTS)
- Database connectors  
- PDF report generators  
- Secure messaging tools  
- UI component libraries  

### Economic Comparison
Over 4 years: Build = $14,000,000, Buy = $14,000,000.  
Build is better strategically because we own the IP and can customize the system.

## Section 4: Risk Assessment

**Technical Risks:**
- AI may not reach 90% accuracy  
- Cairo servers may be insufficient  
- Integration across 4 sites is complex

**Mitigation Strategy:**
- Spiral Model for AI testing  
- Budget $2M for Cairo server upgrade  
- Hybrid lifecycle lets each site progress independently
